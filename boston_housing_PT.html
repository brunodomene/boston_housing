Implementação: Definir uma Métrica de Desempenho¶

difcil medir a qualidade de um modelo dado sem quantificar seu desempenho durante o treinamento e teste. Isso geralmente feito utilizando algum tipo de mtrica de desempenho, atravs do clculo de algum tipo de erro, qualidade de ajuste, ou qualquer outra medida til. Para este projeto, voc ir calcular o coeficiente de determinao, R2, para quantificar o desempenho do seu modelo. O coeficiente de determinao uma estatstica til no campo de anlise de regresso uma vez que descreve o quo "bom" a capacidade do modelo em fazer estimativas.

Os valores para R2 tm um alcance de 0 a 1, que captura a porcentagem da correlao ao quadrado entre a estimativa e o valor atual da varivel alvo. Um modelo R2 de valor 0 sempre falha ao estimar a varivel alvo, enquanto que um modelo R2 de valor 1, estima perfeitamente a varivel alvo. Qualquer valor entre 0 e 1 indica qual a porcentagem da varivel alvo (ao utilizar o modelo) que pode ser explicada pelos atributos. Um modelo pode dar tambm um R2 negativo, que indica que o modelo no melhor do que aquele que estima ingenuamente a mdia da varivel alvo.

Dica: Quando explicar a tcnica de busca em matriz, tenha certeza que voc explicou o motivo dela ser usada, o que a 'matriz' significa nesse caso e qual o objetivo da tcnica. Resposta: Essa tcnica consiste em colocar os hiperparmetros do modelo de predio em uma tabela. Feito isso, o modelo treinado e validado com os dados de treinamento e de validao crusada, gerando uma pontuao para cada combinao de hiperparmetros usados, deste modo, teremos como escolher o melhor conjunto de hiperparmetros para aquele conjunto de dados, para tal, s escolher a linha da tabela que resultou em uma maior pontuao. Um exemplo dessa tcnica seria utiliza-la para obter os melhores hiperparmetros do modelo SVM, que tem os hiperparmetros Kernel e gama, onde para cada um deles se escolhe uma gama de valores a serem testados, esses valores so colocados em uma tabela e o modelo testado para cada combinao desses valores, e escolhendo, por fim, o conjunto que se sai melhor para aquele conjunto de dados.

Questão 8 - Validação Cruzada¶

  • O que a tcnica de treinamento de validao-cruzada k-fold?
  • Quais benefcios essa tcnica proporciona para busca em matriz ao otimizar um modelo?

Dica: Lembre-se de expllicar o que significa o 'k' da validao-cruzada k-fold, como a base de dados dividida e quantas vezes ela executada.

Assim como h um raciocnio por trs de utilizar um conjunto de teste, o que poderia dar errado ao utilizar busca em matriz sem um conjunto de validao cruzada? Voc pode utilizar a documentao para basear sua resposta.

Resposta: Essa tcnica divide os dados de treinamento em k vezes, pega uma parte dessa diviso e usa esses dados para realizar a validao do seu modelo e o resto dessa diviso para treinar, faz essa operao recursivamente at que todos os "pedaos" de dados gerados na diviso sejam utilizados para a validao. Assim, se consegue realizar a validao crusada do modelo preditivo que est sendo utilizado sem que se tenha que realizar a validao com dados que o modelo j "viu", o que possibilita a utilizao da tcnica grid search, pois sem a validao crusada no teria como testar o modelo na grid search com dados desconhecidos j que no permitido utilizar os dados de teste para tal. Sem essas tcnicas o modelo utilizado pode sofrer de overfitting, ou seja, ele acaba decorando os dados, desta forma, ele vai muito bem no treinamento mas muito mal em dados que ele nunca viu. Isso seria evitado caso fosse realizada a validao cruzada junto com o grid search.
